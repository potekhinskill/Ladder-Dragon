Safety first

This project is not investment advice. DRY and Testnet are the defaults. Mainnet LIVE requires BOT_LIVE_CONFIRMED=YES, an exact reviewed commit, a passed preflight, and manual supervision.

Install locally

git clone https://github.com/potekhinskill/Ladder-Dragon.git
cd Ladder-Dragon
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e '.[test,dashboard]'
cp .env.example .env
PYTHONPATH=. pytest -q

How the service works

  1. bin/ai_supervisor.py reads market data, indicators, policy, and risk limits.
  2. bin/autosize_universal.py validates Binance filters and submits only approved plans.
  3. The order journal records intent before a request; lost acknowledgements and restarts reconcile by client and exchange IDs.
  4. OCO/STOP, FIFO lots, fees, and net PnL are recorded in SQLite.

Dashboard

Start with python -m bin.run_dashboard. Open Ladder Dragon or dashboard help. The UI supports localized labels, persistent language selection, balances, open orders, positions, AI quality, host health, logs, and backup metadata.

Raspberry Pi

Follow the Raspberry runbook. Updates use an exact commit:

sudo bash deploy/update_raspberry_pi.sh update <40-character-SHA>

Security

  • Keep `.env`, `.env.dashboard`, keys, tokens, databases, backups, and real logs out of Git.
  • Use a separate read-only Binance key for the dashboard.
  • Review `/backups/` and `/logs/` only through Basic Auth.
  • RAG is local retrieval, not DeepSeek fine-tuning, and cannot modify Risk Manager.
