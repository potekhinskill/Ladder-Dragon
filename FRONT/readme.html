binance_bot — Полное руководство

Windows (PowerShell/WSL), macOS/Linux, Raspberry Pi, автозапуск и безопасность ключей. Версия продукта 2.0.0: «толстые» ордера, Auto-CAP, умная очистка (TTL), режим x10 (2 монеты), отчёты PnL (reports/), интеграция с Pi Dashboard.

⚠️
Важное: не инвестиционная рекомендация. По умолчанию DRY (без реальных ордеров). Реальная торговля только с флагом --live. Проверяйте флаги, пары и лимиты биржи (tickSize/stepSize/minNotional).

Что нового

  • Толстые BUY — выбираются ближайшие уровни ниже рынка и ставятся «толстые» заявки (больше вероятность исполнения).
  • Auto-CAP — динамический лимит на ордер от свободного USDT; мягкие границы --cap-floor-usdt, --cap-ceil-usdt.
  • Smart cleanup (TTL) — двойной TTL: «ближние» к рынку сохраняем, «далёкие/старые» отменяем.
  • Режим x10 — фокус на 2 монетах, 4–5 одновременных BUY на символ, короткие TTL.
  • PNL — репорты в reports/, исправлен InventoryFIFO и 24h лимиты API.
  • Pi Dashboard — сводка/PNL/исполнения, поддержка symbols=, подписанные запросы для equity.

Как работает

  1. bin/ai_supervisor.py (планировщик): получает рынок, считает MA/ATR, применяет PPBS, формирует план, делает умную очистку и Auto-CAP, запускает исполнителя.
  2. bin/autosize_universal.py (исполнитель): нормализация tickSize/stepSize, «толстые» лимитные BUY, OCO по остаткам, контроль minNotional.
  3. Защита от дублей: lock-файл и проверка уже запущенных процессов.

Примечание «Молчаливые» флаги (supervisor-only) не пробрасываются в процесс исполнителя — это нормально и предотвращает рассинхрон CLI.

Установка

python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
chmod +x bin/supervisor_ctl.sh

Быстрый старт (x10, 2 монеты)

./bin/supervisor_ctl.sh start \
  --singleton \
  --symbols SOLUSDT,ETHUSDT \
  --ladder-mode pct \
  --ladder-pct "SOLUSDT=-3,-6,-10,-15;ETHUSDT=-2.5,-5.5,-9" \
  --rolling-ladder --ppbs-guard --auto-oco-holdings \
  --ma-filter 21,50 --ma-kind ema --ma-interval 1h --ma-only-uptrend \
  --atr-window 14 --atr-interval 1h \
  --atr-mult-tp1 2.0 --atr-mult-tp2 3.5 --atr-mult-sl 1.5 \
  --tp1-min 0.08 --tp1-max 0.40 --sl-max 0.12 \
  --max-oco-per-symbol 8 --only-new-fills \
  --status-interval 1 --child-loop-minutes 5 --interval-seconds 60 \
  --auto-cap --alloc-pct 0.90 --cap-floor-usdt 25 --cap-ceil-usdt 80 \
  --target-buy-per-symbol 4 \
  --use-bnb-fees 1 \
  --live

Интеграция с Pi Dashboard

  • /api/trades/summary?hours=24&symbols=ETHUSDT,SOLUSDT — агр. сводка, equity PnL и «Прибл. общая стоимость» по выбранным монетам.
  • /api/trades/filled?hours=24&limit=5000[&symbols=...] — исполненные ордера (комиссии нормализованы в USDT).
  • /api/trades/symbols?hours=168 — список пар за период (для фильтра на фронте).

Ключи Binance берутся из окружения сервиса бота или .env (поддерживается автопоиск).

.env и окружение

Обязательные/основные

BINANCE_API_KEY=...
BINANCE_API_SECRET=...
# Optional:
BINANCE_API_BASE=https://api.binance.com
LOG_LEVEL=INFO

Финансы/комиссии/бюджет

# Комиссии:
USE_BNB_FOR_FEES=1               # платить BNB (скидка)
BOT_FEE_PCT=0.00075              # 0.075% — для оценок там, где биржа не вернула fee

# Auto-CAP / allocation:
BOT_CAP_PER_ORDER=40             # используется планировщиком, может изменяться динамически
BOT_SMART_ALLOC_PCT=0.90         # доля свободного USDT под работу
BOT_SMART_MIN_USDT=20            # минимум, при котором запускается торговля
🛡️
Безопасность секретов: храните .env локально с правами 600 (Linux/macOS) или ограниченными ACL (Windows). На бирже используйте суб-аккаунт, права только SPOT TRADE, без вывода, по возможности — IP-whitelist.

Автозапуск

systemd (Linux)

[Unit]
Description=Binance Bot Supervisor
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=bot
WorkingDirectory=/home/bot/apps/binance_bot
EnvironmentFile=-/home/bot/apps/binance_bot/.env
ExecStart=/bin/bash -lc 'source .venv/bin/activate && ./bin/supervisor_ctl.sh start --singleton ... --live'
Restart=always
RestartSec=6
StandardOutput=append:/home/bot/apps/binance_bot/supervisor.log
StandardError=append:/home/bot/apps/binance_bot/supervisor.log

[Install]
WantedBy=multi-user.target

FAQ и безопасность

  • [MONITOR] Finished. — завершена итерация по инструменту.
  • Отключился интернет? — ордера остаются на бирже; цикл продолжится после восстановления.
  • Наткнулись на лимиты биржи? — проверьте minNotional, увеличьте CAP или уменьшите число ступеней.
  • Большие отклонения PnL? — используйте symbols= в Dashboard, включите подписанные запросы (ключи в окружении), сверяйте остатки «на сейчас».
  • Стоп-кнопка: ./bin/supervisor_ctl.sh stop-all. Логи: supervisor.log, pnl.log.
