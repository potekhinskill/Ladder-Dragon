  • Service: pi-healthd.service on 127.0.0.1:8081
curl -s http://127.0.0.1:8081/api/health | python3 -m json.tool

Read-only API

  • GET /api/health — host time, kernel, CPU, memory, disk, mounts, services, uptime, and network.
  • GET /api/history?hours=N — chart samples from metrics.
  • GET /api/trades/summary?hours=24 — net, realized, cash-flow, and equity metrics.
  • GET /api/trades/filled?hours=24 — sanitized filled-order rows.
  • GET /api/ai/status — AI state, decisions, budget, RAG, and attribution.

All API routes are protected by dashboard authentication. Raw journal and trading methods are not exposed.

Trading and accounting

The dashboard is read-only. It displays the venue/mode banner, balances, reserve, CAP, FIFO positions, average entry, current value, unrealized PnL, open orders, OCO/STOP state, gap-watchdog, reconciliation delta, and recent fills.

Net PnL includes fees and exact attribution. Unresolved fills are excluded. A persistent circuit halt must be reviewed manually before reset.

Environment

  • BOT_STATS_DB — SQLite trade ledger.
  • BINANCE_API_BASE — selected Binance endpoint.
  • DASHBOARD_BINANCE_API_KEY/SECRET — separate read-only account key.
  • AI_RUNTIME_STATUS_FILE — safe heartbeat without secrets.
  • DASHBOARD_FOLLOW_BOT_PATHS=1 — follow the active Testnet/Mainnet paths.

Protected backups and logs

/backups/ exposes only encrypted archives, checksums, and safe inventory after Basic Auth. /logs/ exposes sanitized rotating logs; keys, tokens, authorization headers, and signatures are redacted.

sudo systemctl start ladder-dragon-backup.service
sudo journalctl -u ladder-dragon-backup.service -n 50 --no-pager
