API эндпоинты (системные)

  • GET /api/health — сводка: время, ядро, t°, память, диск, монтирования, сервисы, аптайм, сеть.
  • GET /api/history?hours=N — точки для графиков из metrics.ndjson.
  • GET /api/bot/logs/stream — Server-Sent Events с онлайн-логами mybot.service.

Торговля: сводка и PnL

Эндпоинт возвращает агрегаты за окно и «чистый» PnL по equity (все монеты + USDT). Поддерживает фильтр по инструментам.

  • GET /api/trades/summary?hours=24[&symbols=ETHUSDT,SOLUSDT]
  • Поля: total_trades, buy_volume_usdt, sell_volume_usdt, fees_usdt, cashflow_pnl_usdt, realized_pnl_usdt, equity_pnl_usdt (= net_pnl_usdt), equity_now_usdt, equity_then_usdt, equity_pct, equity_method (balances+klines / db-holdings-minima), equity_assets.

Примеры

curl -sS 'http://127.0.0.1:8081/api/trades/summary?hours=24' \
| python3 -m json.tool | sed -n '1,60p'
curl -sS 'http://127.0.0.1:8081/api/trades/summary?hours=24&symbols=ETHUSDT,SOLUSDT' | python3 - <<'PY'
import sys,json; d=json.load(sys.stdin)
print('method=',d.get('equity_method'))
print('equity_now_usdt=',d.get('equity_now_usdt'))
print('equity_pnl_usdt=',d.get('equity_pnl_usdt'))
print('assets=',d.get('equity_assets'))
PY
Примечания: комиссия учитывается из fee_quote (если есть), иначе оценивается по BOT_FEE_PCT (оплата BNB со скидкой). Проценты в дашборде показываются как equity_pnl_usdt / equity_then_usdt, при слишком малом базисе процент скрывается.

Торговля: исполненные ордера

  • GET /api/trades/filled?hours=24&limit=5000[&symbols=...]
  • Алиасы: /api/orders/filled, /api/fills
  • Поля строки: time (ms), symbol, side, price, qty, quoteQty, commission (в USDT), commissionAsset=USDT
  • Формат в UI: цена — 2 знака после запятой, количество — 4 знака, суммы — 2 знака.
curl -sS 'http://127.0.0.1:8081/api/trades/filled?hours=24&limit=5' | python3 -m json.tool

Символы

GET /api/trades/symbols?hours=168 — уникальные пары за период (для фильтра).

curl -sS 'http://127.0.0.1:8081/api/trades/symbols?hours=168' | python3 -m json.tool

Почему «Прибл. общая стоимость» может отличаться

  • Метод: если equity_method = balances+klines, берутся реальные балансы и свечи Binance — это точно. Если db-holdings-minima, это восстановление по БД (депозиты до начала истории не учитываются, отрицательные остатки обнуляются).
  • Фильтр: используйте symbols= чтобы считать только по монетам бота.
  • Комиссии в BNB: учитываются как уменьшение BNB (если комиссия пришла в BNB).

Окружение (переменные)

  • BOT_STATS_DB — путь к базе с трейдами (пример: /home/bot/apps/binance_bot/db/bot_stats.db).
  • BOT_FEE_PCT — процент комиссии для оценки, по умолчанию 0.00075 (0.075% c BNB-скидкой).
  • BINANCE_API_BASE — хост Binance API (по умолчанию https://api.binance.com).
  • DASHBOARD_BINANCE_API_KEY, DASHBOARD_BINANCE_API_SECRET — отдельный read-only ключ только в .env.dashboard. Торговый .env и окружение процесса бота дашборд не читает.
  • AI_RUNTIME_STATUS_FILE — безопасный heartbeat бота без ключей; стандартный путь /run/mybot/ai_status.json.
  • DASHBOARD_FOLLOW_BOT_PATHS=1 — читать активные Testnet/Mainnet SQLite и AI usage log из heartbeat, чтобы интерфейс всегда соответствовал реально запущенному контуру.

Проверка и запуск вручную

Проверить сумму SHA256

# Имена уже связаны через "latest":
curl -sk https://bot.local/backups/pi-config-latest.sha256.txt -o /tmp/safe.sha256
SAFE=/var/www/bot/backups/pi-config-safe-latest.tgz
sha256sum "$SAFE" | diff - /tmp/safe.sha256 && echo "OK"

Сделать бэкап сейчас

sudo systemctl start make-pi-backup.service
journalctl -u make-pi-backup -n 50 --no-pager

Восстановление (кратко)

  1. Проверить сумму: sha256sum pi-config-safe-TS.tgz | diff - pi-config-TS.sha256.txt
  2. Развернуть SAFE: sudo tar -xzf pi-config-safe-TS.tgz -C /
  3. Развернуть SECRETS: sudo tar -xzf /path/pi-config-secrets-TS.tgz -C /
  4. Создать venv в /home/bot/apps/binance_bot и установить пакеты из extras/pip-freeze-*.txt
  5. sudo systemctl daemon-reload && sudo systemctl enable --now mybot pi-healthd nginx
Полная инструкция лежит внутри архива: extras/RESTORE.md.
